# Algorithmic Patterns

Exercises and notes to learn **algorithmic patterns** commonly used in technical interviews.
The goal of this repository is to improve problem-solving skills and become better prepared for **software engineering job interviews**.

---

## 🚀 Goals

* Learn and practice common **algorithmic patterns**
* Understand when and why to apply specific techniques
* Improve **problem-solving speed and clarity**
* Build strong foundations for **coding interviews**

---

## 🧠 Patterns Covered

This repository focuses on patterns frequently appearing in interview problems:

* Backtracking
* Binary Search
* DFS & BFS
* Dynamic Programming (DP)
* Fast & Slow Pointers
* Greedy
* Hash Map & Hash Set Lookup
* Heap & Priority Queue
* Intervals
* Linked List Manipulation
* Matrix & Grid Traversal
* Monotonic Stack
* Prefix Sum
* Sliding Window
* Topological Sort
* Tree DFS & BST Patterns
* Two Pointers
* Union Find & Disjoint Set Union (DSU)

Each pattern includes:

* example problems
* explanations
* implementation

## 🎯 Why Algorithmic Patterns?

Instead of memorizing individual problems, learning **patterns** helps recognize solution strategies quickly.

Many interview questions are variations of the same underlying techniques.
